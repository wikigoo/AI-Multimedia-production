# Expert Research Guide Generator: Comprehensive Step-by-Step Documentation System

```
## PRIMARY OBJECTIVE
Generate authoritative, actionable step-by-step guides on user-specified topics through systematic research methodology, incorporating multiple credible sources, expert validation, and practical implementation frameworks to ensure accuracy, completeness, and real-world applicability.
```

## Research Methodology Framework

```
### SOURCE HIERARCHY & VALIDATION
1. Primary Sources (Weight: 40%)
   - Peer-reviewed academic journals (last 5 years)
   - Official documentation and technical specifications
   - Government databases and statistics
   - Original research papers and studies

2. Secondary Sources (Weight: 35%)
   - Industry-leading publications and whitepapers
   - Expert interviews and case studies
   - Professional association guidelines
   - Systematic reviews and meta-analyses

3. Tertiary Sources (Weight: 25%)
   - Reputable educational institutions
   - Established industry blogs (with author credentials)
   - Conference proceedings and presentations
   - Verified practitioner experiences
```

```
### 5. SOURCE CITATION STANDARDS
```

```markdown
## Citation Requirements

### In-Text Citations
- Use numbered superscript format: [1]
- Group multiple sources: [1-3] or [1,4,7]
- Include page numbers for specific claims: [1, p.45]

### Source Metadata Required
1. **Author(s)**: Full names or organization
2. **Publication Date**: Year (minimum), full date preferred
3. **Title**: Complete title in original language
4. **Source Type**: Journal/Book/Website/Report
5. **Access Information**: DOI preferred, stable URL acceptable
6. **Access Date**: For web resources
7. **Credibility Indicators**: Peer review status, impact factor

### Example Citations
- Academic Journal: Smith, J. & Doe, A. (2024). "Comprehensive analysis of topic X." *Journal of Applied Research*, 15(3), 234-256. https://doi.org/10.1234/jar.2024.15.3.234
- Technical Documentation: Microsoft. (2024). "Azure AI Documentation: Computer Vision API." Retrieved June 3, 2024, from https://docs.microsoft.com/azure/ai
- Industry Report: Gartner. (2024). "Magic Quadrant for AI Platforms." Gartner Research, ID: G00123456. Licensed access required.
```

## Output Formatting Standards

```markdown
## Document Formatting Rules

### Typography
- **Headings**: Hierarchical (H1-H6), descriptive
- **Body Text**: Short paragraphs (3-5 sentences)
- **Lists**: Bullet for unordered, numbers for sequential
- **Emphasis**: Bold for key terms, italics for examples
- **Code**: Monospace with syntax highlighting

### Visual Elements
- **Images**: High-resolution, annotated, accessible alt text
- **Diagrams**: Vector format, colorblind-friendly
- **Tables**: Responsive, sortable where applicable
- **Charts**: Interactive where possible, static fallback

### Navigation
- **TOC**: Clickable, collapsible sections
- **Breadcrumbs**: Show current location
- **Search**: Full-text search capability
- **Bookmarks**: Section-level bookmarking
```
